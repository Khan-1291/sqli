🔒 Implementation Details 
# SECURE: Parameterized LIKE query
results = db.execute(
    "SELECT * FROM products WHERE name LIKE ?",
    ('%' + query_param + '%',)
).fetchall()

Even with wildcards in the parameter, the ? placeholder ensures the entire value is treated as a literal string. A UNION payload like ' UNION SELECT... will be searched as a product name, not executed as SQL.

📋 Security Best Practices
  • Always use parameterized queries — never concatenate user input into SQL
  • Use ORM safely — even ORMs can be vulnerable if raw queries are used
  • Input validation — whitelist expected formats (but don't rely on it alone)
  • Least privilege — database accounts should have minimal required permissions
  • WAF/IPS — Web Application Firewalls provide defense-in-depth
  • Code review — regularly audit for SQL injection vulnerabilities
